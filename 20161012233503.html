Python的Sqlite3模块导入失败，初次尝试

Posted by dreamChou24 on 三 12 十月 2016

sqlite3自从python2.5开始就自动内嵌在Python的标准库中。但是我在centos6.5中直接import sqlite3确报错了。报错信息如下

>>> import sqlite3
    Traceback (most recent call last):
      File "<stdin>", line 1, in <module>
        File "/usr/local/lib/python2.7/sqlite3/__init__.py", line 24, in <module>
          from dbapi2 import *
        File "/usr/local/lib/python2.7/sqlite3/dbapi2.py", line 28, in <module>
          from _sqlite3 import *
      ImportError: No module named _sqlite3

事实上是虽然sqlite3在Python标准库中了，但是sqlite数据库并没有集成在centos6.5的操作系统中。因此还需要在操作系统中安装。 使用yum命令可以很快地安装。

yum -y install sqlite-devel

但是这个时候还有另一个问题，那就是centos6.5默认python是2.6版本，并且yum依赖于此python2.6。因此使用yum命令安装的sqlite数据实际上会在python2.6上的生成.so文件。

[root@MiWiFi-R3L-srv local]# find / -name _sqlite3.so
/usr/lib64/python2.6/lib-dynload/_sqlite3.so

而我的操作系统还安装着另一个版本的Python2.7,因此暂时我想到的办法是做软连接或者直接就拷贝一个到python2.7的lib-dynload目录下。那么之后再用python2.7的virtualenv命令创建虚拟环境时， .so文件也会被引入virtualenv。 如果已经存在virtualenv，那么也可以拷贝一个到virtualenv下。

[root@MiWiFi-R3L-srv local]# cp /usr/lib64/python2.6/lib-dynload/_sqlite3.so ~/movie/lib/python2.7/lib-dynload/
[root@MiWiFi-R3L-srv local]# cd 
[root@MiWiFi-R3L-srv ~]# cd movie/
[root@MiWiFi-R3L-srv movie]# source bin/activate
(movie) [root@MiWiFi-R3L-srv movie]# python
Python 2.7.8 (default, Oct  9 2016, 08:25:27) 
    [GCC 4.4.7 20120313 (Red Hat 4.4.7-17)] on linux2
    Type "help", "copyright", "credits" or "license" for more information.
    >>> import sqlite3

那么现在就可以初始化一个sqlite数据库了。

#coding:utf-8

import sqlite3

conn = sqlite3.connect('test.db')
print "connect ok"
conn.execute('''
        CREATE TABLE COMPANY(
            ID INT PRIMARY KEY NOT NULL,
            NAME TEXT NOT NULL,
            AGE INT NOT NULL,
            ADDRESS CHAR(50),
            SALARY REAL);''')
print "init tables done"
conn.execute("INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY) \
              VALUES (1, 'Paul', 32, 'California', 20000.00 )");
conn.execute("INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY) \
              VALUES (2, 'Allen', 25, 'Texas', 15000.00 )");
conn.commit()
print "insert data done"
conn.execute("UPDATE COMPANY set SALARY = 25000.00 where ID=1")
conn.commit
print "update data done"
conn.execute("DELETE from COMPANY where ID=2;")
conn.commit
print "delete data done"
cursor = conn.execute("SELECT id, name, address, salary  from COMPANY")
for row in cursor:
    print "ID = ", row[0]
    print "NAME = ", row[1]
    print "ADDRESS = ", row[2]
    print "SALARY = ", row[3], "\n"
conn.close()
print "disconnect ok"

执行以上脚本，会发现脚本边上会有一个test.db的二进制文件。而这也就是创建的sqlite数据库了。
